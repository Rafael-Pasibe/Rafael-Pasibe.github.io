VISUAL NOVEL DEVELOPMENT — SKILL TRANSLATION GUIDE
===============================================

PURPOSE
-------
Teach people how to APPLY their existing skills to Visual Novel (VN) development,
not how to refine or relearn those skills.

This framework focuses on TRANSLATION:
“How does what I already do become usable VN content?”

-----------------------------------------------
CORE IDEA
-----------------------------------------------
A Visual Novel is a DELIVERY SYSTEM for creative work.

It combines:
- Narrative structure
- Timed presentation
- Asset coordination
- Simple logic

Most contributors plug into asset creation and orchestration,
not complex programming or game mechanics.

-----------------------------------------------
THE VN PIPELINE (TEACH THIS FIRST)
-----------------------------------------------
1. Story & structure are defined
2. Assets are created
3. Assets are named & organized
4. Assets are triggered by script
5. VN engine displays/plays them
6. Player input advances the state

Once people understand this pipeline,
they immediately see where their role fits.

-----------------------------------------------
ROLE TRANSLATION GUIDE
-----------------------------------------------

PRODUCT MANAGERS / PRODUCERS
----------------------------
Not: “Learn game design”
Instead: “Coordinate creative delivery”

Teach:
- Chapter-based milestone planning
- Asset dependency tracking
- Scene completion criteria
- Creative bottleneck management

Outputs:
- Scene lists
- Asset checklists
- Naming conventions
- Release schedules

Key realization:
“A VN is content-heavy but logic-light.”

-----------------------------------------------

SOFTWARE DEVELOPERS
-------------------
Not: “Become a game engine expert”
Instead: “Glue assets together”

Teach:
- Asset triggering
- State flags & conditions
- Branching logic
- Text flow control

Outputs:
- Scripts
- Condition checks
- Save/load logic
- UI behavior tweaks

Key realization:
“VN scripting is orchestration, not traditional programming.”

-----------------------------------------------

ARTISTS (CHARACTER / BG / UI)
-----------------------------
Not: “Learn new art skills”
Instead: “Deliver modular assets”

Teach:
- Sprite expressions & poses
- Background reuse strategies
- UI-safe composition
- Aspect ratios & transparency

Outputs:
- Layered PNGs
- Expression sets
- UI elements
- CG event art

Key realization:
“One drawing becomes many in-game moments.”

-----------------------------------------------

MUSICIANS / COMPOSERS
---------------------
Not: “Learn game audio theory”
Instead: “Compose for emotional states”

Teach:
- Loop-safe composition
- Mood tagging (calm, tension, romance)
- Scene-length neutrality
- Fade-in/fade-out awareness

Outputs:
- Looping tracks
- Stingers
- Theme variations

Key realization:
“VN music supports pacing, not gameplay.”

-----------------------------------------------

SOUND DESIGNERS
---------------
Not: “Learn middleware deeply”
Instead: “Provide reusable cues”

Teach:
- UI sound effects
- Ambient loops
- Narrative punctuation

Outputs:
- Labeled SFX packs
- Volume-normalized clips
- Short, clean effects

Key realization:
“Small sounds carry narrative weight.”

-----------------------------------------------

VOICE ACTORS
------------
Not: “Learn game acting”
Instead: “Record modular dialogue”

Teach:
- Line-by-line recording
- Emotional consistency
- File naming conventions
- Text-sync pacing

Outputs:
- Clean WAV files
- Per-line recordings
- Alternate takes

Key realization:
“Your voice is a selectable asset, not a linear performance.”

-----------------------------------------------
TEACH INTERFACES, NOT TOOLS
-----------------------------------------------
Avoid teaching specific engines first.

Instead teach:
- Where each role’s output plugs in
- Required formats
- How assets are triggered

Key framing:
“You don’t need to know the engine,
you need to know how your work is CALLED.”

-----------------------------------------------
USE TRANSLATION EXERCISES
-----------------------------------------------
Avoid generic skill drills.

Use:
- “Turn one character into a VN-ready asset set”
- “Create three looping tracks tagged by emotion”
- “Record a VN dialogue exchange with file naming”

Focus on application, not mastery.

-----------------------------------------------
SHARED CONSTRAINTS (EVERYONE LEARNS)
-----------------------------------------------
- File naming rules
- Folder structure
- Version control basics
- How changes affect other roles

This builds cross-discipline respect.

-----------------------------------------------
CORE TEACHING PHRASE
-----------------------------------------------
Repeat often:

“Visual Novels are about coordination, not complexity.”
